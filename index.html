Prototype · 2025

RAG Knowledge Base

Problem

Organizations store years of knowledge in wikis, PDFs, and internal tools. When someone needs an answer, they search manually, ask a colleague, or give up. The knowledge exists. It's just not accessible.

Solution

A pipeline that indexes any document collection and makes it queryable in plain language. Ask a question, get a precise answer with the source it came from. No search, no digging.

how it works
01
Documents any format
Drop .md, .txt, or .pdf files into the data/ folder. The pipeline is document-agnostic — wikis, contracts, specs, HR policies, anything.
02
Indexing LlamaIndex
LlamaIndex chunks documents and builds a retrieval index. Runs once on ingest — no re-indexing unless documents change.
03
Vector Storage ChromaDB + HuggingFace
HuggingFace embeddings stored locally in ChromaDB. Persistent across sessions. No API cost for embedding.
04
API Endpoint FastAPI
REST endpoint accepts natural language questions. Retrieves the most relevant document chunks before passing to the LLM.
05
LLM Inference Groq · Llama 3.3-70b
Groq for speed. Llama 3.3-70b synthesizes retrieved context into a precise answer. Free tier sufficient for PoC.
06
Answer + Source traceable output
Returns the answer plus the source document chunks it drew from. Traceable, verifiable — not a black box.
built in under a day. deployable as internal tool or client PoC.
