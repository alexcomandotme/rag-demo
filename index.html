how it works
01
Documentsany format
Drop .md, .txt, or .pdf files into the data/ folder. Document-agnostic — wikis, contracts, specs, HR policies, anything.
02
IndexingLlamaIndex
Chunks documents and builds a retrieval index. Runs once on ingest — no re-indexing unless documents change.
03
Vector StorageChromaDB + HuggingFace
Local embeddings stored in ChromaDB. Persistent across sessions. No API cost for embedding.
04
API EndpointFastAPI
REST endpoint accepts natural language questions. Retrieves relevant chunks before passing to the LLM.
05
LLM InferenceGroq · Llama 3.3-70b
Groq for speed. Synthesizes retrieved context into a precise answer. Free tier sufficient for PoC.
06
Answer + Sourcetraceable output
Returns the answer plus source chunks. Traceable, verifiable — not a black box.
built in under a day. deployable as internal tool or client PoC.
